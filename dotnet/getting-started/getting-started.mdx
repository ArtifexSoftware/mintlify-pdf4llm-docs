# Getting started

This page walks you from a blank .NET project to a working PDF-to-Markdown conversion. It assumes no prior knowledge of MuPDF.NET.

---

## Prerequisites

- [.NET SDK](https://dotnet.microsoft.com/download) 5.0 or later (or any .NET Framework / .NET Standard 2.0-compatible toolchain)
- A PDF file to test with — any PDF works

---

## 1. Create a project

If you already have a project, skip to step 2.

```bash
dotnet new console -n MyRagApp
cd MyRagApp
```

---

## 2. Install PDF4LLM

```bash
dotnet add package PDF4LLM
```

This installs PDF4LLM and its dependency, MuPDF.NET, automatically. You do not need to install MuPDF.NET separately.

<Callout type="warning">
**Assembly conflict?** If you see _"An assembly with the same simple name 'PDF4LLM' has already been imported"_, your installed version of MuPDF.NET already bundles PDF4LLM. Remove the explicit `PDF4LLM` package reference and use the one bundled with MuPDF.NET. See [Requirements & compatibility](/introduction/requirements) for the full explanation.
</Callout>

---

## 3. Write your first conversion

Replace the contents of `Program.cs` with the following:

```csharp
using MuPDF.NET;
using PDF4LLM;

// Open the PDF
Document doc = new Document("document.pdf");

// Convert to Markdown
string markdown = PdfExtractor.ToMarkdown(doc);

doc.Close();

// Print the result
Console.WriteLine(markdown);
```

Run it:

```bash
dotnet run
```

You should see the PDF's content printed to the console as GitHub-compatible Markdown — headings prefixed with `#`, tables rendered as Markdown tables, bold and italic text preserved.

---

## 4. Save the output to a file

Rather than printing to the console, write the Markdown to disk:

```csharp
using MuPDF.NET;
using PDF4LLM;
using System.IO;

Document doc = new Document("document.pdf");
string markdown = PdfExtractor.ToMarkdown(doc);
doc.Close();

File.WriteAllText("output.md", markdown);
Console.WriteLine("Saved to output.md");
```

---

## 5. Choose an output format

`ToMarkdown` is the right default for most LLM and RAG use cases, but PDF4LLM offers two other formats depending on your needs.

**Plain text** — same layout analysis, no Markdown syntax. Useful when your chunker or downstream system doesn't handle Markdown well.

```csharp
string text = PdfExtractor.ToText(doc);
```

**JSON** — full layout structure with bounding boxes, block types, and page dimensions. Useful when you need to inspect document structure or build a custom output format.

```csharp
string json = PdfExtractor.ToJson(doc);
File.WriteAllText("layout.json", json);
```

See [Core concepts](/introduction/core-concepts) for a detailed comparison of when to use each format.

---

## 6. Process specific pages

By default, every page is processed. Pass a `pages` list (zero-based) to restrict processing:

```csharp
// First three pages only
string markdown = PdfExtractor.ToMarkdown(
    doc,
    pages: new List<int> { 0, 1, 2 }
);
```

---

## 7. Open a document from a file path directly

You can pass a file path string directly to `ToMarkdown` without constructing a `Document` object first:

```csharp
string markdown = PdfExtractor.ToMarkdown("document.pdf");
```

This is convenient for one-off conversions. For cases where you are processing the same document in multiple ways — extracting Markdown and form fields, for instance — open a `Document` once and pass it to each method to avoid re-parsing the file.

```csharp
// Efficient: open once, call multiple methods
Document doc = new Document("document.pdf");

string markdown = PdfExtractor.ToMarkdown(doc);
string text     = PdfExtractor.ToText(doc);
var fields      = PdfExtractor.GetKeyValues(doc);

doc.Close();
```

---

## What you have

At this point you have a working conversion. Here is what `PdfExtractor` did under the hood:

1. Parsed the PDF's internal structure using MuPDF
2. Analysed each page spatially to determine reading order, column layout, and block types
3. Identified headings by font size and weight
4. Detected tables and rendered them with Markdown pipe syntax
5. Preserved bold, italic, and monospaced text formatting
6. Emitted a single Markdown string covering all processed pages

---

## Common patterns at a glance

```csharp
using MuPDF.NET;
using PDF4LLM;

Document doc = new Document("document.pdf");

// Markdown — default for LLM/RAG
string md = PdfExtractor.ToMarkdown(doc);

// Markdown with images embedded as Base64
string mdWithImages = PdfExtractor.ToMarkdown(doc, embedImages: true);

// Markdown with images saved to ./images/
string mdSaveImages = PdfExtractor.ToMarkdown(
    doc,
    writeImages: true,
    imagePath: "./images",
    imageFormat: "png"
);

// Specific pages (0-based)
string mdPages = PdfExtractor.ToMarkdown(doc, pages: new List<int> { 0, 1, 2 });

// With OCR for scanned PDFs
string mdOcr = PdfExtractor.ToMarkdown(doc, useOcr: true, ocrLanguage: "eng");

// Plain text
string text = PdfExtractor.ToText(doc);

// JSON layout
string json = PdfExtractor.ToJson(doc);

// Structured object model
ParsedDocument parsed = PdfExtractor.ParseDocument(doc);

// LlamaIndex-compatible loading
var reader    = PdfExtractor.LlamaMarkdownReader();
var llamaDocs = reader.LoadData("document.pdf");

// Form fields
var fields = PdfExtractor.GetKeyValues(doc);

doc.Close();
```

## Troubleshooting

**`FileNotFoundException` when opening a document**
The path passed to `new Document(...)` or `ToMarkdown(...)` must be absolute, or relative to the working directory at runtime. For `dotnet run`, that is the project root. Either use an absolute path or copy your PDF into the project folder.

**Output is empty or very sparse**
Some PDFs store page content as rasterised images, which is common in scanned documents. If the returned Markdown is blank or contains only whitespace, enable OCR:

```csharp
string markdown = PdfExtractor.ToMarkdown(doc, useOcr: true);
```

See the [OCR guide](/guides/ocr) for language options and performance considerations.

**Tables are not rendering correctly**
Complex tables with merged cells or non-standard layouts may not convert perfectly. Use `ToJson` to inspect the raw layout data and determine whether the table is being detected as a table block or as separate text blocks.

**Assembly conflict on startup**
See the warning in step 2, and full details in [Requirements & compatibility](/introduction/requirements).
